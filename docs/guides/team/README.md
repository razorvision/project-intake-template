## Key Processes

### New Team Member?
1. Start with [Onboarding Checklist](ONBOARDING_CHECKLIST.md)
2. Get assigned an onboarding buddy
3. Complete Day 1 setup tasks

### Code Changes
1. Create branch per [Branch Strategy](BRANCH_STRATEGY.md)
2. Submit PR for review
3. Address feedback per [Code Review Guidelines](CODE_REVIEW_GUIDELINES.md)
4. Merge after approval

### Documentation
1. Follow [Documentation Guidelines](DOCUMENTATION_GUIDELINES.md)
2. Keep docs updated with code changes
3. Use templates where provided

### Team Interaction
1. Follow [Code of Conduct](CODE_OF_CONDUCT.md)
2. Give constructive feedback in reviews
3. Communicate proactively about blockers
